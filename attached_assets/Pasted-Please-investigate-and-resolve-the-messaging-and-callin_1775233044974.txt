When a message is sent to a team member, it should be delivered to the intended recipient

The sender also receives the notification, which suggests the message or notification routing is incorrect
Please fix this so messages are sent to the correct user and notifications are delivered to the proper recipient
2. Notification and message visibility issue

Please also investigate the issue where:

the notification is received
but the actual message cannot be located in the conversation

This means the alert may be triggering, but the message is either:

not being stored correctly
not being displayed in the right conversation
or not syncing properly in the interface

Please review the full messaging flow, including:

message delivery
conversation mapping
notification logic
storage and retrieval
real-time sync behavior
3. Calling issue

Please investigate the same type of issue for calls as well.

If a call is placed to another user, it should reach the correct recipient

Users should be able to update the content of a sent message
